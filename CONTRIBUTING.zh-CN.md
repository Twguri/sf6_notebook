# 参与贡献 SF6 Notebook

感谢你对 **SF6 Notebook（街霸6 玩家备忘录）** 的关注与兴趣 ❤️  
这是一个以**个人使用 + 实战工具**为核心定位的开源项目，欢迎任何符合项目理念的贡献。

---

## 🧭 项目理念（请在贡献前阅读）

在提交 Issue 或 Pull Request 之前，请理解并尊重以下原则：

- **本地优先（Local-first）**  
  用户数据应始终保存在用户本地（浏览器）。不引入账号系统、服务器或云同步依赖。

- **工具导向，而非平台导向**  
  这是一个服务于玩家训练与复盘的工具，而不是社区、内容平台或社交网络。

- **实用性第一**  
  功能应直接帮助：对策准备、连段整理、对局复盘或操作输入效率。

- **保持克制与简单**  
  优先接受小而清晰的改动，避免过度抽象或复杂架构。

---

## 🐛 Bug 反馈

如果你发现了 Bug，请：

1. 先查看 **Issues**，确认问题是否已经被报告。
2. 如果没有，请新建 Issue，并尽量包含以下信息：
   - 问题的清晰描述
   - 复现步骤
   - 期望行为 vs 实际行为
   - 使用的浏览器 / 系统（桌面或手机）
   - 截图或录屏（如适用）

---

## 💡 功能建议 / 新想法

非常欢迎合理的功能建议，尤其是以下方向：

- 对策 / 连段工作流优化
- 格斗键盘（Fighter Input）体验改进
- 手机端可用性与交互优化
- 性能、可读性或可访问性改进

提交功能建议时，请尽量说明：

- 这个功能解决了什么实际问题
- 为什么它符合 SF6 Notebook 的定位
- 你对交互或 UI 的初步想法（如有）

## 🔧 开发环境搭建

### 1️⃣ Fork 并克隆仓库

```bash
git clone https://github.com/<你的用户名>/sf6_notebook.git
cd sf6_notebook
```

### 2️⃣ 安装依赖

```bash
npm install
```

### 3️⃣ 启动本地开发

```bash
npm run dev
```

---

## 🔀 Pull Request 指南

- 请为你的修改新建分支：
  ```bash
  git checkout -b feature/your-feature-name
  ```

- 建议一个 PR 只做一件事情（一个功能或一个修复）。
- 尽量保持与现有代码风格一致。
- 如涉及 UI / 交互改动，请附上截图或简要说明。
- 破坏性改动（Breaking changes）请先通过 Issue 讨论。

---

## 📱 移动端注意事项

SF6 Notebook 对手机端使用体验非常重视。在贡献 UI 或交互相关改动时，请特别注意：

- 触控目标尺寸不少于 **44px**
- 处理好安全区（刘海、底部 Home 指示条）
- 软键盘弹出时的布局与遮挡问题
- 避免强依赖键盘快捷键（手机无法使用）
