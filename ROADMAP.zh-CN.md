# SF6 Notebook 项目路线图

本文档描述 **SF6 Notebook（街霸6 玩家备忘录）** 的规划方向与优先级。  
路线图以「**实用工具优先、玩家视角**」为核心原则，不承诺具体时间点，而强调逐步、稳定的改进。

---

## 🎯 指导原则

所有规划内容遵循以下原则：

- **本地优先（Local-first）**：不引入强制账号系统或服务器依赖
- **玩家实用性优先**：功能必须直接服务于对局准备与复盘
- **小步迭代**：优先做高收益的小改动，而非大规模重构
- **跨平台演进**：Web → 桌面端 → 移动端

---

## ✅ 当前状态

- 基础备忘录工作流（Matchup / Combo / Notes）
- 格斗键盘输入系统（Normal / Fighter 模式）
- 浏览器本地存储（支持导出 / 导入）
- 帧数表功能（部分角色 / 数据仍在补充）
- 中英文双语界面
- GitHub Pages 部署

---

## 🚧 短期规划（近期迭代）

### 📱 手机端可用性优化
- 扩大触控目标尺寸，降低误触率
- 改善软键盘弹出时的布局与遮挡问题
- 输入时自动隐藏或调整悬浮按钮（如 Back to Top）
- 优化小屏设备下的弹窗（Modal）行为

### ⌨️ 格斗键盘（Fighter Input）增强
- 常用方向 / 按键的一键插入按钮
- 当前输入模式（Normal / Fighter）的清晰视觉提示
- 改善移动端光标与文本编辑体验

### 📊 帧数表完善
- 补全缺失的角色与招式数据
- 提升不同角色之间的数据一致性
- 标注随版本变动的关键帧数差异

---

## 🧪 中期规划

### 🌐 PWA 支持
- 支持「添加到主屏幕」
- 离线访问已保存的备忘录内容
- 应用图标、启动页与主题色优化

### 📦 数据管理能力
- 更细粒度的导出 / 导入（按角色 / 连段组）
- 导入时的冲突处理机制
- 辅助性的本地备份方案（仍保持本地优先）

### 🎨 UI / UX 打磨
- 长时间使用场景下的字体与排版优化
- 降低密集视图中的视觉噪音
- 大数据量情况下的性能优化

---

## 🚀 长期规划

### 📱 原生移动端（Android）
- 基于 Capacitor 构建 Android APK
- 更好地控制软键盘与输入行为
- 可选的触觉反馈（Haptic Feedback）

### 💻 桌面端版本
- 基于 Tauri 构建 Windows 可执行文件（.exe）
- 原生窗口与快捷键支持
- 离线优先的桌面使用体验

---

## ❌ 明确不做的方向（Out of Scope）

以下内容**明确不在计划内**：

- 用户账号 / 登录系统
- 云端同步或中心化服务器
- 社交功能（点赞、评论、分享）
- 商业化、广告或订阅机制

---

## 🤝 社区参与

路线图并非一成不变，欢迎理性讨论。

如果你的想法：
- 能提升训练效率
- 简化对策或连段整理流程
- 在不增加复杂度的前提下改善手机体验
